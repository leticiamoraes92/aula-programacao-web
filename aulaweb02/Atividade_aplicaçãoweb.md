# 📘 Atividade – Análise de Aplicações Web de Receitas

## 🔎 1. Pesquisa de Aplicações Web

### 🍲 TudoGostoso
- Plataforma colaborativa de receitas.
- Grande volume de conteúdo com receitas enviadas por usuários.
- Possui sistema de avaliações e comentários.

**Pontos fortes:**
- Grande variedade de receitas  
- Alto tráfego e reconhecimento  
- Interação entre usuários  

**Pontos fracos:**
- Qualidade inconsistente  
- Receitas repetidas  

---

### 🥘 Cookpad
- Plataforma global focada em receitas caseiras.
- Funciona como uma rede social culinária.

**Pontos fortes:**
- Forte engajamento da comunidade  
- Busca por ingredientes  
- Facilidade de navegação  

**Pontos fracos:**
- Conteúdo repetitivo  
- Interface simples  

---

### 🍳 Panelinha
- Site com receitas testadas e conteúdo educativo.
- Foco em qualidade e confiabilidade.

**Pontos fortes:**
- Alta qualidade das receitas  
- Conteúdo confiável  

**Pontos fracos:**
- Menor quantidade de receitas  

---

## 🧱 2. Estrutura das Aplicações Web

### 🔹 Frontend (Interface)
- Página inicial com receitas em destaque  
- Barra de busca  
- Listagem de receitas com filtros  
- Página de detalhes (ingredientes + preparo)  
- Perfil do usuário  
- Login/cadastro  

---

### 🔹 Backend (Servidor)
- API (REST ou GraphQL)  
- Banco de dados de receitas e usuários  
- Sistema de autenticação  
- Lógica de busca e filtros  

---

### 🔹 Banco de Dados
- Armazena receitas, usuários e comentários  
- Otimização de buscas  

---

### 🔹 CMS (Administração)
- Gerenciamento de receitas  
- Controle de usuários e conteúdo  

---

### 🔹 Recursos Extras
- Avaliações e comentários  
- Notificações  
- Integração com redes sociais  

---

## 🔄 3. Fluxo de Navegação

1. Usuário acessa o site  
2. Visualiza receitas em destaque  
3. Pesquisa por nome ou ingrediente  
4. Seleciona uma receita  
5. Visualiza detalhes  
6. Pode salvar, comentar ou avaliar  
7. Usuário logado pode publicar receitas  

---

## 🗣️ 4. User Stories

- **User Story 1:**  
Como visitante, eu quero pesquisar receitas por nome ou ingrediente para encontrar facilmente o que desejo preparar.

- **User Story 2:**  
Como usuário, eu quero visualizar o passo a passo da receita para preparar corretamente.

- **User Story 3:**  
Como visitante, eu quero visualizar receitas por categorias para encontrar opções rapidamente.

---

## 🧩 5. Wireframe – Kitchen Boss

### 📌 Estrutura:

**🔹 Header:**
- Logo do site  
- Barra de pesquisa  
- Menu (Assados, Sopas, Massas, Bebidas)  
- Menu lateral  

**🔹 Conteúdo Principal:**
- Mensagem de boas-vindas  
- Cards com categorias (imagens)  

**🔹 Footer:**
- Termos de uso  
- Política de privacidade  
- Suporte  
- Quem somos  

---

### 🎯 Objetivos do Wireframe
- Organização clara das informações  
- Navegação simples  
- Facilidade de busca  
- Boa experiência do usuário  
- Estrutura responsiva  

---

## ✅ Conclusão

A atividade permitiu:
- Entender a estrutura de aplicações web  
- Definir requisitos com User Stories  
- Planejar a navegação do sistema  
- Criar um protótipo inicial (wireframe)  

Essas etapas ajudam a evitar erros no desenvolvimento e melhoram a experiência do usuário.
