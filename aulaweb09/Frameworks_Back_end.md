
## 📌 1. O que é um Framework Back-end
Framework back-end é um conjunto de ferramentas e padrões que facilitam o desenvolvimento do lado do servidor (APIs, banco de dados e regras de negócio).

**Vantagens:**
- Código organizado  
- Segurança integrada  
- Facilidade de manutenção e escalabilidade  

**Comparação:**
- Sem framework → mais código manual, maior risco de erros  
- Com framework → produtividade e padronização  

---

## 📌 2. Características dos Frameworks
- Arquitetura organizada (MVC ou camadas)  
- Componentização (módulos, serviços, middlewares)  
- Programação assíncrona (melhor desempenho)  
- Segurança (proteção contra SQL Injection)  
- Sistema de rotas e APIs  

---

## 📌 3. Exemplos de Frameworks
- Express.js (Node.js) → leve e flexível  
- Django (Python) → completo (full-stack)  
- Laravel (PHP) → sintaxe elegante e produtiva  

**Critérios de escolha:**
- Complexidade do projeto  
- Tamanho da equipe  
- Performance desejada  

---

## 📌 4. Arquitetura e Design Patterns
Arquitetura define a estrutura do sistema.

**Tipos de padrões:**
- Criacionais → criação de objetos (Factory, Singleton)  
- Estruturais → organização (Adapter, Proxy)  
- Comportamentais → comunicação (Observer, Strategy)  

---

## 📌 5. Monólito vs Microsserviços
- Monólito → sistema único, simples, difícil de escalar  
- Microsserviços → dividido em serviços, escalável, mais complexo  

👉 Escolha depende do projeto (não existe melhor absoluto)

---

## 📌 6. Express.js
Framework para Node.js usado para criar APIs e servidores.

**Características:**
- Roteamento simples  
- Uso de middlewares  
- Leve e rápido  
- Ideal para APIs REST  

---

## 📌 7. Fluxo de Requisição
1. Usuário interage com o front-end  
2. Navegador envia requisição HTTP  
3. Servidor processa  
4. Acessa banco ou API  
5. Retorna resposta (JSON)  
6. Front-end atualiza a tela  

---

## 📌 8. Métodos HTTP
- GET → buscar dados  
- POST → criar dados  
- PUT/PATCH → atualizar  
- DELETE → remover  

---

## 📌 9. API REST
API permite comunicação entre sistemas.

**Características REST:**
- Stateless (sem memória de requisições)  
- Uso de HTTP  
- Dados em JSON  
- Uso de endpoints (URLs)  

---

## 📌 10. Criando API com Express
Passos básicos:
1. Criar projeto  
2. Instalar Express (`npm install express`)  
3. Criar arquivo (ex: api.js)  
4. Rodar servidor (`node api.js`)  
5. Criar rotas  

---

## 📌 11. Deploy com Render
- Hospedagem em nuvem  
- Integração com GitHub  
- Deploy automático  
- Ideal para APIs e projetos acadêmicos  

---

## 📌 12. Atividade
- Criar API com rota de data/hora  
- Fazer deploy no Render  
- Criar frontend consumindo a API  
- Subir tudo no GitHub  

---

## ✅ Conclusão
Frameworks back-end:
- Facilitam o desenvolvimento  
- Melhoram organização e segurança  
- Ajudam na criação de APIs escaláveis  

👉 Express.js é uma ótima escolha para iniciar com APIs REST.
