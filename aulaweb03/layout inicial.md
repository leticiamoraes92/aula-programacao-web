# 📚 Aula 03 – Introdução ao Versionamento e Deploy

## 🧩 Assunto abordado em aula

A aula abordou conceitos fundamentais de **versionamento de software**, uso do **Git** e o processo de **deploy**, essenciais no desenvolvimento de sistemas.

---

## 🔹 📌 Versionamento

O versionamento é o processo de controlar e registrar as alterações feitas em um código ao longo do tempo.

### ✅ Benefícios:
- 👥 Trabalho em equipe sem conflitos  
- 🕒 Histórico de alterações  
- 🔄 Possibilidade de reverter versões  
- 📊 Organização e rastreabilidade  

---

## 🔹 🔢 Versionamento Semântico (SemVer)

Padrão de versionamento:

- 🚨 **MAJOR (2.0.0)** → mudanças incompatíveis  
- ✨ **MINOR (1.1.0)** → novas funcionalidades  
- 🛠️ **PATCH (1.0.1)** → correção de erros  

---

## 🔹 💻 Git (Controle de Versão)

O Git é uma ferramenta utilizada para versionar código.

### 🔧 Permite:
- 📌 Registrar alterações (commit)  
- 🔄 Sincronizar com repositórios (GitHub)  
- 👨‍💻 Trabalho colaborativo  

---

## 🔹 🌿 Branches e Merge

- 🌿 **Branch:** ramificação do código  
- 🏠 **Main:** versão principal estável  
- 🚀 **Feature:** nova funcionalidade  
- 🩹 **Hotfix:** correção urgente  

🔀 **Merge:** união das alterações de diferentes branches  

---

## 🔹 🚀 Deploy

Deploy é o processo de disponibilizar a aplicação para os usuários.

### 🌍 Ambientes:
- 💻 Desenvolvimento  
- 🧪 Testes (Staging)  
- 🚀 Produção  

---

## 🔹 ☁️ Hospedagem (Vercel)

- 🔗 Integração com GitHub  
- ⚡ Deploy automático  
- 🌐 Publicação rápida de aplicações  

---

## 📝 ✅ Resumo

A aula apresentou os conceitos de versionamento, uso do Git, criação de branches, versionamento semântico e deploy, mostrando a importância dessas práticas no desenvolvimento de software moderno.
